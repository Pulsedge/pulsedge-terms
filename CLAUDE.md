# CLAUDE.md

Site estático (HTML puro, sem build) com checkout/assinatura e páginas legais do Pulsedge, hospedado no Vercel em pulsedge.com.br.

- `pages/`: As 4 páginas HTML do site, agrupadas numa pasta. As URLs públicas continuam limpas (`/home`, `/termos`, `/privacidade`, `/assinar`) via `rewrites` no `vercel.json` — não são os nomes de arquivo nem caminhos físicos.
  - `home.html`: Página institucional (`/home`), destino do redirect de `/`. Hero navy com slogan, banda de estatísticas agregadas (`GET https://api.pulsedge.com.br/stats` → empresas/municípios monitorados, histórico total), seção "O que fazemos" e transparência do fundador — sem preço ou CTA de cadastro (acesso é só por convite/link pessoal). Nav no header para Termos de Serviço/Política de Privacidade.
  - `assinar.html`: Página de checkout (`/assinar?h={lead_links.id}`). Valida o link via `GET https://api.pulsedge.com.br/subscribe?l={id}`, renderiza os 2 cards de plano (Semanal/Diário) com preço e link de pagamento 100% dinâmicos vindos da API (`weekly_plan_price`/`daily_plan_price`, `weekly_plan_link_url`/`daily_plan_link_url`; `PLAN_META` guarda só nome/badge/features estáticos), exige checkbox de aceite de Termos/Privacidade para habilitar o botão de pagamento, mostra um FAQ em acordeão (`<details>`) sobre cancelamento/segurança/dados antes do rodapé, e mostra tela de erro "link não disponível" em caso de 422/falha. Sem nav no header — página de conversão de propósito único, sem links de saída além dos já presentes no checkbox de consentimento. `noindex` no `<head>` — não deve aparecer em buscas.
  - `termos.html`: Termos de Serviço — foco B2B (exclui CDC), cap de responsabilidade na última mensalidade paga, sem reembolso por cancelamento do assinante (só por indisponibilidade técnica da Pulsedge). Nav no header para Início/Privacidade.
  - `privacidade.html`: Política de Privacidade/LGPD — dados de assinantes (execução de contrato) vs. dados de terceiros/empresas monitoradas via CETESB/Receita Federal (legítimo interesse/dado público, Art. 7º VIII e IX), direitos do titular, contatos. Nav no header para Início/Termos.
- `vercel.json`: `redirects` (`/` → `/home`, e força `www.pulsedge.com.br` → `pulsedge.com.br` apex) e `rewrites` que mapeiam as URLs limpas (`/home`, `/termos`, `/privacidade`, `/assinar`) para os arquivos físicos em `pages/`.
- `robots.txt` / `sitemap.xml`: Na raiz (convenção exigida por crawlers). `robots.txt` bloqueia `/assinar`; `sitemap.xml` lista `/home`, `/termos`, `/privacidade`.
- `favicon.png`: Ícone navy (256x256, otimizado) referenciado via `<link rel="icon">` nas 4 páginas.
- `README.md`: Passo a passo de deploy manual no Vercel (histórico, não reflete fluxo de dev atual).
- `tests/`: Suíte Playwright (E2E) cobrindo as 4 páginas — header dinâmico/scroll do `home.html`, contadores, modal do avatar, mapa de SP, header sólido de `termos`/`privacidade`, e o fluxo completo de `assinar.html` (planos, consentimento, FAQ, estados de erro/expirado) com a API mockada via `page.route`. `static-server.js` imita os `rewrites` do `vercel.json` apontando para `pages/*.html`. Pasta própria com seu próprio `package.json` (não afeta o build estático do Vercel — excluída via `.vercelignore`). Rodar com `cd tests && npm install && npx playwright test`.

## Fluxo de trabalho
Antes de finalizar qualquer mudança: rode a suíte Playwright (`cd tests && npx playwright test`) e confirme que está tudo passando. Se a mudança alterar ou adicionar comportamento (novo fluxo, validação, estado de erro, elemento de UI testável), crie ou atualize specs em `tests/*.spec.js` cobrindo o caso.

## Padrão visual
Header navy (`--navy: #1a2b4a`) com logo, fonte Inter para corpo/UI + Lora (serif) para headlines e números de destaque — mesmo pareamento tipográfico dos templates de e-mail. Mesmas CSS vars (`--bg`, `--accent`, `--radius`, `--section-bg`) repetidas inline em cada HTML (sem CSS compartilhado/build step). Nav institucional em botões-pílula (Início/Termos de Serviço/Política de Privacidade) no header de `home.html`, `termos.html` e `privacidade.html` — propositalmente ausente em `assinar.html` para não distrair do CTA de pagamento. Seções com fundo alternado (`--section-bg`) e badges numerados circulares replicam o estilo dos e-mails transacionais. Rodapé padronizado nas 4 páginas, espelhando os templates de e-mail: `© {ano dinâmico via JS} Pulsedge — operado por Vinícius Pereira Barbosa · CNPJ · Piracicaba, SP, Brasil`.
